# VE Database ML 도입 완료 보고서

> **프로젝트**: 건설 VE(Value Engineering) 종합 분석 대시보드  
> **작업일**: 2026-05-06  
> **대상 데이터**: 727개 VE 대안, 9개 프로젝트, 210 KG 노드, 619 KG 엣지  

---

## 1. 도입 배경

기존 AI VE 자문 시스템의 핵심 한계는 **키워드 매칭 기반 검색**이었습니다.

```
기존 방식:
사용자 질의 → cube_taxonomy.py의 SPACE_KEYWORDS(24개) + HOW2_KEYWORD_MAP(~150개)
             → 정확히 일치하는 키워드가 있을 때만 KG Hop 탐색
             → 없으면? → 결과 0건
```

이 방식은 "옥상 방수"처럼 정확한 키워드를 알고 입력해야만 동작하며, "지붕 물 새는 문제"나 "에너지 효율 개선" 같은 자연어 질의에는 전혀 대응할 수 없었습니다.

## 2. 도입된 ML 기법 (4개 Tier)

### Tier 1 — Embedding 기반 시맨틱 검색

| 항목 | 상세 |
|---|---|
| **기법** | Sentence Embedding + 코사인 유사도 |
| **모델** | `paraphrase-multilingual-MiniLM-L12-v2` (384차원) |
| **학습 방식** | 사전학습 모델(Pre-trained) — 별도 파인튜닝 없이 바로 사용 |
| **데이터** | 727개 대안의 결합 텍스트 (제안명 + 원안설명 + 대안설명 + 분류정보) |
| **검색 방식** | 질의 텍스트 → 384차원 벡터 → 727개 대안 벡터와 내적(코사인) → Top-K |

```
작동 원리:
"지붕 물 새는 문제"  →  [0.12, -0.34, 0.78, ...]  (384차원 벡터)
                            ↓  코사인 유사도
"옥상 방수 공법 변경"  →  [0.15, -0.31, 0.81, ...]  → 유사도 0.835 ✅
"LED 조명 교체"       →  [0.45, 0.22, -0.11, ...]  → 유사도 0.12  ✗
```

> **핵심 개선**: 키워드가 정확히 일치하지 않아도 **의미적으로 유사한 대안**을 찾아냅니다.

### Tier 2 — Hybrid RAG (Embedding + KG + Gemini)

| 항목 | 상세 |
|---|---|
| **기법** | Retrieval-Augmented Generation (RAG) |
| **구성** | 시맨틱 검색(Tier 1) + Knowledge Graph 구조 매칭 + Gemini 2.0 Flash |
| **복합 스코어** | `0.6 × Semantic_Score + 0.4 × KG_Score` |

```
Step 1: Embedding Search → Top 20 후보 (의미 기반)
Step 2: KG Hop 매칭 → Space/HOW2 구조적 관련성 점수
Step 3: 복합 스코어 재정렬 → 최종 Top 10
Step 4: (선택) 상위 5개 대안 + 원문 → Gemini Prompt → 자연어 분석 답변
```

> **시맨틱 검색은 "의미"를 잡고, KG는 "구조(공간/공종)"를 잡습니다.**
> 둘을 결합하면 단독보다 정확합니다.
> 예: "옥상 방수" → 시맨틱으로 방수 관련 대안 20개 → KG에서 "옥상" Space + "방수코킹공사" HOW2 매칭 → 대안-191이 1위로 올라감.

### Tier 3 — 자동 분류 ML (TF-IDF + Calibrated SVM)

| 항목 | 상세 |
|---|---|
| **기법** | TF-IDF 벡터화 + CalibratedClassifierCV(LinearSVC) |
| **학습 데이터** | 727개 라벨링된 VE 대안 (제안명 + 설명 → HOW1/HOW2/가치유형) |
| **HOW1 대분류** | 523 samples, 6 classes → 교차검증 정확도 **58.0%** |
| **HOW2 세부분류** | 486 samples, 33 classes → 교차검증 정확도 **40.5%** |
| **ValueType 분류기** | 727 samples, 4 classes → 교차검증 정확도 **66.9%** |

```
작동 원리:
"옥상 방수를 우레탄에서 액체방수로 변경"
    ↓ TF-IDF (5000 features, bigram)
    ↓ CalibratedClassifierCV(LinearSVC)
    → HOW1: A 건축공사 (confidence: 64.0%) ✅
    → HOW2: A-08 방수코킹공사 (confidence: 35%) — 확정
    → ValueType: 가치혁신형 (confidence: 70.8%) ✅
```

#### Tier 3 보완 사항 (v2 개선)

1단계 평가에서 **HOW2 분류기 38% 정확도가 불가용 수준**이라는 지적에 대응하여 다음 3가지를 개선했습니다:

| 개선 항목 | 기존 (v1) | 개선 (v2) |
|---|---|---|
| **분류 계층** | HOW2 단일 (40클래스) | HOW1 대분류(6) + HOW2 세부(33) 2단계 |
| **신뢰도 제어** | 없음 (랜덤 수준도 출력) | **임계값 35%** — 미달 시 "불확실" 표시 |
| **후보 제공** | 1등만 반환 | **Top-3 후보** 항상 반환 |
| **확률 보정** | 소프트맥스 근사 | **CalibratedClassifierCV** (실제 확률) |
| **HOW2 정확도** | 38.0% | **40.5%** (+2.5%) |
| **ValueType 정확도** | 63.3% | **66.9%** (+3.6%) |

**HOW1 대분류가 6클래스로 58% 정확도**를 달성하여, HOW2가 불확실해도 "어느 공사 분야인지"는 신뢰 가능합니다.

분류 결과 예시:
```
"LED 조명기구를 에너지 절감형으로 교체"
  HOW1: D 전기공사 (77.4%) ✅         ← 6클래스라 정확도 높음
  HOW2: 불확실 — Top3: D-05(33%), D-09(26%), F-03(2%)
        ↑ 후보 2개가 모두 전등설비 관련 → 참고 가능

"엘리베이터 설치 위치 변경"
  HOW1: E 기계설비공사 (60.7%) ✅
  HOW2: 불확실 — Top3: E-11(20%), B-06(13%), F-03(11%)
        ↑ 확정 불가하지만 1순위가 정확
```

|---|---|---|---|---|

| 2 | 방수코킹 / 전체 / 비용절감형 | 108건 | +0.17% | +13.7% |
| 3 | 철근콘크리트 / 구조체 / 가치혁신형 | 141건 | +0.01% | +50.9% |
| 4 | 정보통신설비 / 도로 / 성능강조형 | 50건 | +1.20% | +16.4% |
| 5 | 배관설비 / 전체 / 가치혁신형 | 86건 | 0% | +43.3% |
| 6 | 창호 / 계단실 / 가치혁신형 | 82건 | +0.11% | +30.9% |

## 3. Gemini 프롬프트 엔지니어링 (v2 개선)

1단계 평가에서 **환각(Hallucination) 제어 전략 부재** 지적에 대응하여 프롬프트를 전면 개선했습니다.

| 개선 항목 | 기존 | 개선 |
|---|---|---|
| **시스템 제약** | 없음 | 4개 환각 방지 규칙 삽입 |
| **도메인 지식** | 없음 | CUBE 분류체계(WHERE/HOW/가치유형) 주입 |
| **temperature** | 1.0 (기본) | **0.3** (사실 기반 강화) |
| **top_p** | 1.0 (기본) | **0.8** (분산 축소) |
| **max_tokens** | 무제한 | **1200** (간결성 확보) |
| **불확실성 처리** | 없음 | "데이터에서 확인되지 않음" 명시 요구 |

### 추가된 시스템 제약

```
[시스템 제약 — 반드시 준수]
1. 제공된 VE 대안 데이터만 근거로 답변. 데이터에 없는 방법/기법/제품 제안 금지.
2. 비용/성능 수치는 제공된 데이터 값만 인용. 추정치 생성 금지.
3. 불확실한 정보는 "데이터에서 확인되지 않음" 명시.
4. 대안 번호 반드시 명시하여 원본 데이터 추적 가능하게 함.
```

### 추가된 도메인 지식 주입

```
## 건설 VE 분류체계 (CUBE 온톨로지)
- 관련 공간(WHERE): 옥상, 지하층, ...
- 관련 공종(HOW): 방수코킹공사, 철근콘크리트공사, ...
- 가치유형 정의: 가치혁신형(성능↑비용↓), 비용절감형(성능=비용↓), ...
```

## 6. 대시보드 UI 변경

### 6.1 AI VE 자문 페이지 (05. AI VE 자문)

**기존 (Before)**:
- 단순 텍스트 입력 + 검색 버튼
- KG Hop 키워드 매칭만 가능
- 결과: 대안명 + KG 경로 나열

**개선 (After)**:
- 검색 모드 선택: `Hybrid (Embedding+KG)` / `Embedding Only` / `KG Legacy`
- Gemini AI 분석 답변 토글 (체크 시 자연어 비교분석 생성)
- 각 결과에 **유사도 진행 바** (시맨틱 점수 + KG 점수 표시)
- Hybrid RAG 시각화: 시맨틱 매칭은 점선, KG 매칭은 실선
- 검색 소요 시간 실시간 표시

### 6.2 대안 상세 모달
- **유사 대안 (Embedding 기반)** 섹션 추가
- 해당 대안과 벡터 유사도가 높은 5개 대안을 자동 추천
- 클릭 시 해당 대안 상세로 바로 이동

### 6.3 Overview 페이지 (01. 종합 현황)
- **ML Cluster Insights** 섹션 추가 (하단)
- 8개 클러스터 카드: 크기, 자동 라벨, 대표 대안, 평균 절감율/가치변화, 공종 분포 태그

## 7. 기술 아키텍처 (최종)

```
사용자 질의
    │
    ├──→ [Tier 1] sentence-transformers MiniLM-L12-v2
    │         │
    │         ▼
    │    727×384 벡터 인덱스 ──→ 코사인 유사도 ──→ Top 20 후보
    │
    ├──→ [Tier 2] Hybrid RAG
    │         │
    │         ├── Top 20 + KG(210노드, 619엣지) Space/HOW2 매칭
    │         ▼
    │    복합 스코어 (0.6×Sem + 0.4×KG) ──→ 최종 Top 10
    │         │
    │         └── (선택) Gemini 2.0 Flash ──→ 자연어 분석 답변
    │                    temperature=0.3, 환각 방지 제약
    │
    ├──→ [Tier 3] 자동 분류
    │         │
    │         ├── HOW1 (6클래스, 58.0%) ── CalibratedClassifierCV
    │         ├── HOW2 (33클래스, 40.5%) ── 신뢰도 임계값 35% + Top-3
    │         └── ValueType (4클래스, 66.9%)
    │
    └──→ [Tier 4] K-Means 클러스터링 (k=8)
              │
              └── 8개 클러스터 ── 자동 라벨 + 대표 대안 + 인사이트
```
