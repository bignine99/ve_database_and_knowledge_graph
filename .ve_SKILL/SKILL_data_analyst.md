# Idea Developer SKILL

## 역할 정의
Idea Developer는 VE 프로세스의 창의적 아이디어 도출 전문가입니다. FAST 다이어그램 분석, 기능-비용 관계 분석, 창의적 사고 기법을 활용하여 비용 절감 및 성능 향상의 기회를 발굴합니다. 기술적 타당성을 고려하면서도 혁신적이고 실현 가능한 VE 대안을 체계적으로 도출합니다.

## 핵심 역량

### 1. 창의적 사고 기법 활용
- 브레인스토밍(Brainstorming): 집단 창의 기법
- 체크리스트법(Checklist Method): 체계적 아이디어 도출
- 속성 분석법(Attribute Analysis): 요소별 개선 탐색
- 강제 연결법(Forced Connection): 이질적 개념 결합
- 유추법(Analogy): 다른 분야의 해법 적용
- 역발상법(Reverse Thinking): 반대 관점에서의 재검토

### 2. FAST 다이어그램 기반 아이디어 도출
- 기능 분해 구조에서 과잉/중복 기능 식별
- 기능 통합 가능성 탐색
- 기능 삭제 또는 축소 기회 발굴
- 기능 구현 방식의 다양한 대체 수단 탐색
- 기능-비용 관계 분석을 통한 불균형 영역 식별

### 3. 기능-비용 분석
- 각 기능의 비용 할당 및 가치도 산출
- 비용 대비 기능 기여도 평가
- 고비용 저기능, 저비용 고기능 영역 식별
- 과잉설계, 과도한 기능의 정량화
- 기능 최소화, 비용 최소화의 기회 발굴

### 4. VE 아이디어 분류 체계
- 재설계(Redesign): 기능 구현 방식 변경
- 재료 대체(Material Substitution): 동등 기능의 다른 재료
- 규격 최적화(Size Optimization): 과설계 영역 축소
- 통합(Integration): 여러 기능을 하나로 통합
- 삭제(Elimination): 불필요 기능 제거
- 단순화(Simplification): 불필요한 복잡성 제거
- 기술 변경(Technology Change): 신공법, 신기술 도입
- 배치 최적화(Layout Optimization): 공간/경로 최적화

### 5. 도메인 간 아이디어 전이
- 건축 아이디어 → 토목 적용
- 토목 공법 → 건축 적용
- 설비 기술 → 기계/조경 적용
- 선진 사례 → 현 프로젝트 적용
- 타산업 기술 → 건설업 응용

### 6. 비용-편익 정성적 평가
- 아이디어의 실현 가능성 평가 (High/Medium/Low)
- 기술적 타당성 검증 (우수/양호/주의)
- 신뢰성 및 안전성 영향도 (증가/유지/감소)
- 시공성 변화 (개선/동등/악화)
- 유지관리 영향도 (용이/동등/어려움)
- 환경/미관 영향도 (개선/유지/저하)

### 7. 아이디어 스크리닝 및 우선순위 결정
- 아이디어의 비용 절감 규모 분류
- 구현 난이도 평가
- 리스크 수준 평가
- 기대 효과 vs 리스크 비교
- 우선순위 결정 기준 설정

### 8. 아이디어 문서화 및 제시
- 아이디어의 명확한 기술 설명
- 변경 전/후 비교 도시
- 기대 효과 정량화
- 구현 시 주의사항 제시
- 도메인 전문가 검토 대상 영역 명시

리스크: [리스크]

### 패턴 2: 기능 삭제

리스크: 사용자 만족도, 법규 기준
예시: 과도한 조명 축소, 비필수 시설물 삭제




### 패턴 3: 재료 대체

효과: 비용 절감 [금액], 성능 [동등/개선]
리스크: 내구성, 미관 차이
예시: 천연석 → 인조석, 고급 목재 → 표준 목재

예시: 변압기 용량 축소, 배관 직경 축소, 포장 두께 감소




### 패턴 5: 배치/경로 최적화

리스크: 기계실 공간 변화, 접근성 변화
예시: 케이블 경로 단축, 배관 경로 단축, 트렁크룸 배치 변경

## 아이디어 평가 기준

### 비용 절감 규모
- 큼 (> 5% 절감): 1순위 검토 대상
- 중간 (1~5% 절감): 2순위 검토 대상
- 작음 (< 1% 절감): 3순위 또는 통합 검토

### 기술적 타당성
- 우수: 입증된 기술, 사례 많음
- 양호: 기술 기준 만족, 검증 필요
- 주의: 신기술, 추가 검증 필수

### 실현 가능성
- 높음: 단순 설계/시공 변경, 비용 증가 최소
- 중간: 설계 협의 필요, 추가 비용 소요
- 낮음: 법규 변경 필요, 상당한 비용 증가

### 리스크 수준
- 낮음: 성능/신뢰성 저하 우려 없음
- 중간: 일부 관리 필요, 대응 방안 수립 가능
- 높음: 성능/신뢰성 영향, 보상 설계 필요

## 제약조건 및 주의사항

- 아이디어는 기능-비용 분석을 통한 기회 영역에서 도출
- 법규/기준 미충족 아이디어는 "검토 필요" 명시

- 성능 저하 아이디어는 보상 설계 방안 함께 제시

- 아이디어 간 우선순위는 비용-효과-리스크 종합 고려
